he::Animation::Animation ( )

Create blank animation with default state

virtual he::Animation::~Animation ( )
virtual

This could happen under two circumstances:

Destroy animation.

  1. Natural Death: Adds child (if any) to the animation loop and calls the callback. Assuming that the callback isn't going to refer to this animation anymore directly or indirectly.
  2. Suicide: Forcibly kills the child (and subsquently the whole family tree) without calling the registered callback (if any)
